# Product

## Register

product

## Users

Modders CK3 (hobbyistes, développeurs de scénarios) qui écrivent des scripts Jomini/CK3 dans l'éditeur de mods. Ils consultent la doc en parallèle de leur éditeur de code, souvent sur un second écran en dark mode. Leur job : trouver le bon pattern rapidement et l'adapter sans se perdre dans la navigation.

## Product Purpose

Référence technique pour les patterns narratifs CK3 réutilisables : story cycles, event chains, scopes, recipes, snippets et checklist de debug. Le site documente des patterns de production, pas des notes personnelles. Succès = l'utilisateur trouve le bon pattern en moins de 10 secondes et peut le copier directement dans son mod.

## Brand Personality

Dense, précis, sombre. Outil premium pour développeurs sérieux. Pas d'excuses, pas de padding superflu : chaque pixel sert le contenu.

Références : Raycast docs, Linear, Wezterm docs.

## Anti-references

- GitBook : sidebar blanche, cards identiques, aucune personnalité.
- MkDocs / Docusaurus par défaut : bleu Bootstrap, navigation plate, look framework générique.
- Medium / blog : typographie éditoriale, pas d'outil technique.

## Design Principles

1. **Le contenu prime sur le chrome.** La structure de navigation doit disparaître quand on lit. Zero décoration non fonctionnelle.
2. **Densité intentionnelle.** Un outil de référence se consulte vite. Pas d'aération excessive : la densité IS la feature.
3. **Cohérence structurelle.** Toutes les pages utilisent l'espace disponible de la même façon. Pas de margin: auto arbitraire.
4. **Copier, pas lire.** Chaque code block est immédiatement copiable. La typographie monospace est first-class.

## Accessibility & Inclusion

WCAG AA. Dark mode par défaut (SRE-style), light mode disponible. Pas d'animations de layout. Pas de dépendance aux couleurs seules pour la sémantique.
